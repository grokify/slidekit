## [v0.2.0] - 2026-02-07

### Highlights

- Cobra CLI with read, plan, apply, create, and serve commands
- MCP server for AI assistant integration with 7 presentation tools
- Shared operations layer (`ops` package) powering both CLI and MCP

### Added

- `slidekit read` command to output presentations in TOON or JSON format ([`8f34fcf`](https://github.com/grokify/slidekit/commit/8f34fcf))
- `slidekit plan` command to compute diff between current and desired state ([`8f34fcf`](https://github.com/grokify/slidekit/commit/8f34fcf))
- `slidekit apply` command to apply changes with `--confirm` requirement ([`8f34fcf`](https://github.com/grokify/slidekit/commit/8f34fcf))
- `slidekit create` command to create presentations from stdin JSON ([`8f34fcf`](https://github.com/grokify/slidekit/commit/8f34fcf))
- `slidekit serve` command to start MCP server on stdio ([`8f34fcf`](https://github.com/grokify/slidekit/commit/8f34fcf))
- MCP `read_deck` tool to read presentations in TOON/JSON format ([`55c4076`](https://github.com/grokify/slidekit/commit/55c4076))
- MCP `list_slides` tool to list slide IDs and titles ([`55c4076`](https://github.com/grokify/slidekit/commit/55c4076))
- MCP `get_slide` tool to retrieve single slide by ID ([`55c4076`](https://github.com/grokify/slidekit/commit/55c4076))
- MCP `plan_changes` tool to compute diff between states ([`55c4076`](https://github.com/grokify/slidekit/commit/55c4076))
- MCP `apply_changes` tool to apply diff with confirmation requirement ([`55c4076`](https://github.com/grokify/slidekit/commit/55c4076))
- MCP `create_deck` tool to create new presentations ([`55c4076`](https://github.com/grokify/slidekit/commit/55c4076))
- MCP `update_slide` tool to update single slide with confirmation ([`55c4076`](https://github.com/grokify/slidekit/commit/55c4076))
- `ops` package with shared logic: `ReadDeck`, `PlanChanges`, `ApplyChanges`, `CreateDeck` ([`f24e1e4`](https://github.com/grokify/slidekit/commit/f24e1e4))
- `ops` package slide operations: `ListSlides`, `GetSlide`, `UpdateSlide` ([`f24e1e4`](https://github.com/grokify/slidekit/commit/f24e1e4))
- Backend registry for managing presentation backends ([`f24e1e4`](https://github.com/grokify/slidekit/commit/f24e1e4))
- `EncodeSlide` method to TOONEncoder for single slide encoding ([`2c1246b`](https://github.com/grokify/slidekit/commit/2c1246b))

## [v0.1.0] - 2026-02-07

### Highlights

- Initial release with canonical slide model, Marp backend, and TOON format

### Added

- Canonical data model: `Deck`, `Section`, `Slide`, `Block`, `Audio`, `Diff` ([`initial`](https://github.com/grokify/slidekit/commit/initial))
- Marp Markdown reader with frontmatter, directives, and speaker notes parsing ([`initial`](https://github.com/grokify/slidekit/commit/initial))
- Marp Markdown writer with lossless round-tripping ([`initial`](https://github.com/grokify/slidekit/commit/initial))
- Backend interface: `Read`, `Write`, `Plan`, `Apply`, `Create` ([`initial`](https://github.com/grokify/slidekit/commit/initial))
- TOON encoder for token-optimized AI output (~8x smaller than JSON) ([`initial`](https://github.com/grokify/slidekit/commit/initial))
- 7 slide layouts: title, title_body, title_two_col, section, blank, image, comparison ([`initial`](https://github.com/grokify/slidekit/commit/initial))
- 7 block types: paragraph, bullet, numbered, code, image, quote, heading ([`initial`](https://github.com/grokify/slidekit/commit/initial))
- Audio metadata support for TTS and video generation ([`initial`](https://github.com/grokify/slidekit/commit/initial))
- Section detection via `_class: section-divider` directives ([`initial`](https://github.com/grokify/slidekit/commit/initial))

### Tests

- 30 test cases covering reader, writer, and TOON encoder ([`initial`](https://github.com/grokify/slidekit/commit/initial))
- Real-world validation with 65-slide production deck ([`initial`](https://github.com/grokify/slidekit/commit/initial))
